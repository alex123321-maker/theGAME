# Backlog (бэклог) M1–M3

## M1 — bootstrap (запуск) проекта

### Цель
Получить запускаемый foundation (базовый) проект с камерой, seed и повторной генерацией.

### Задачи
- [ ] Открыть проект в Godot 4.6.1 и дать движку пересоздать служебные файлы.
- [ ] Проверить autoload: `GameConfig`, `RunContext`, `DebugBus`.
- [ ] Проверить `run/main_scene`.
- [ ] Проверить input map.
- [ ] Запустить сцену bootstrap.
- [ ] Убедиться, что карта рисуется.
- [ ] Убедиться, что `R`, `T`, `Tab`, `G`, `Q`, `E` работают.

### Критерий готовности
Проект запускается, seed виден, карта перегенерируется без ошибок.

---

## M2 — MapData и сериализация

### Цель
Гарантировать, что карта — это данные, которые можно сохранить и восстановить.

### Задачи
- [ ] Прогнать `MapSerializer.to_json_text`.
- [ ] Сохранить JSON-файл для 3 референсных seed.
- [ ] Считать JSON обратно через `MapSerializer.from_json_text`.
- [ ] Сравнить width / height / seed / tile count / entry count / central zone count.
- [ ] Добавить минимальный smoke test (смоук-тест) на совпадение базовых инвариантов.

### Критерий готовности
Повторная загрузка не ломает структуру карты; базовые инварианты совпадают.

---

## M3 — генератор макроформы и маршрутов

### Цель
Получить карту с читаемым центром и 2–4 направлениями подхода.

### Задачи
- [ ] Проверить работу `MacroShapePass`.
- [ ] Проверить работу `RoutesPass`.
- [ ] Прогнать `MapValidator` минимум на 10 seed подряд.
- [ ] Отдельно проверить сиды с малыми и большими числами.
- [ ] Зафиксировать 3 регрессионных seed.

### Критерий готовности
Минимум 10 seed подряд проходят базовую валидацию без soft-lock (софт-лока).
